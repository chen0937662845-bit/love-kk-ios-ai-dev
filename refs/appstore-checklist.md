# App Store 上线前详细检查清单

> 本文件在 C1 节点调用，小K逐项对话确认。
> 每项确认后才进下一项，不跳过。

---

## 第一条：Info.plist 权限描述

**检查项：**
- [ ] 使用了麦克风 → `NSMicrophoneUsageDescription` 已填写
- [ ] 使用了语音识别 → `NSSpeechRecognitionUsageDescription` 已填写
- [ ] 使用了相机 → `NSCameraUsageDescription` 已填写
- [ ] 使用了相册 → `NSPhotoLibraryUsageDescription` 已填写
- [ ] 使用了位置 → `NSLocationWhenInUseUsageDescription` 已填写

**风险**：任何敏感权限缺少描述字符串，Apple 直接拒审，不给解释。

**检查方法**：打开 `ios/App/App/Info.plist`，确认每个使用的权限都有对应 key。

---

## 第二条：隐私政策 + 服务条款

**检查项：**
- [ ] 隐私政策 URL 存在且可访问（不能是 404）
- [ ] 服务条款 URL 存在且可访问
- [ ] App 内有链接入口（通常在设置页或登录页）
- [ ] 链接用 `Browser.open()` 打开外部浏览器，不内嵌 WebView

**风险**：URL 不可访问会被拒审。内嵌 WebView 展示法律文件在某些地区合规风险高。

---

## 第三条：IAP 内购产品

**检查项：**
- [ ] App Store Connect 中产品已创建
- [ ] 产品状态是「准备提交」或「已批准」（不能是草稿）
- [ ] 订阅产品已签署「付费 App 协议」（Agreements 页面）
- [ ] 沙盒测试账号已创建并测试过完整购买流程
- [ ] 产品 ID 和代码里的完全一致（大小写敏感）

**风险**：订阅协议没签是最容易漏掉的，导致订阅功能无法提交审核。

---

## 第四条：第三方登录合规

**检查项：**
- [ ] 有 Google / 微信 / 其他第三方登录 → Apple 登录已同时接入
- [ ] Apple 登录按钮样式符合 HIG 规范（不能自定义成非标准样式）
- [ ] 登录流程在真机上测试通过（模拟器行为和真机不同）

**风险**：有任何第三方登录但缺少 Apple 登录，100% 被拒审。

---

## 第五条：加密合规（出口合规）

**检查项：**
- [ ] 是否自己实现了加密算法？

**如果只用 HTTPS / 系统层 TLS（大多数情况）：**
- App Store Connect 提交时选「None of the algorithms」
- 不需要额外的出口合规文档

**如果自己实现了加密算法：**
- 需要提交加密注册（ERN）
- 在 Info.plist 添加 `ITSAppUsesNonExemptEncryption = YES`

---

## 第六条：数据库写入安全

**检查项：**
- [ ] 所有用户输入写库前过滤 `\x00` 等控制字符
- [ ] 字段长度有上限，超出截断不报错
- [ ] 用户未登录时，写库操作直接 `return`，不执行
- [ ] 错误情况下有 catch，不让用户看到原始错误信息

**风险**：`\x00` 写入 PostgreSQL 会触发神秘报错，很难排查。

---

## 第七条：TestFlight 真机测试

**检查项：**
- [ ] 完整走过登录流程（真机，不是模拟器）
- [ ] 完整走过核心功能流程
- [ ] 完整走过付费流程（沙盒环境）
- [ ] 网络断开时的异常情况测试过
- [ ] 不同 iOS 版本测试过（至少覆盖当前版本和上一个主版本）

**风险**：模拟器和真机行为在登录（特别是 Sign in with Apple）和 IAP 上存在差异，模拟器没问题真机可能挂。

---

## 第八条：App Store Connect 元数据

**检查项：**
- [ ] 截图尺寸符合要求（6.7" / 6.1" / iPad 等）
- [ ] 截图内容真实（不能是空界面或明显假数据）
- [ ] App 描述不包含竞品名称
- [ ] 关键词没有包含他人商标
- [ ] 审核备注（Review Notes）填写了测试账号（如果有登录）
- [ ] 年龄分级填写完整

---

## 所有检查通过后 → 进入 C2 合规扫描
