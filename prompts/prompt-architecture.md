# AI Prompt 架构模板

> 本文件在 A6 节点调用。
> 小K根据主人描述的 AI 功能，填充这个模板，生成 prompt 架构草稿。

---

## 模板结构

```markdown
# Prompt 架构文档：[功能名]

## 一、输出格式定义（先定格式，再写 prompt）

> 这是 prompt 和前端之间的合同，改格式必须两边同步改。

| 字段名 | 类型 | 枚举值 / 范围 | 前端行为 |
|--------|------|--------------|----------|
| [field1] | string | [val_a / val_b / val_c] | [对应 UI 显示逻辑] |
| [field2] | number | [0-100] | [对应进度条 / 颜色] |
| [field3] | boolean | [true / false] | [是否展示某个模块] |

输出 example（prompt 里必须带这个，不能只描述）：
\`\`\`json
{
  "field1": "val_a",
  "field2": 85,
  "field3": true
}
\`\`\`

---

## 二、Step Zero：门槛检查

> 整体情况不符合时，在入口拦住，不进行后续分析。
> 不设这一步，后面所有判断都会产生噪音。

触发条件：[什么情况下整体不符合]

触发后返回：
\`\`\`json
{
  "status": "rejected",
  "reason": "[说明原因]"
}
\`\`\`

---

## 三、判断规则

> 每条规则必须明确——「什么条件 → 输出什么值」。
> 不允许出现「可以理解为」「间接体现了」这类模糊措辞。

| 规则 | 触发条件 | 输出值 |
|------|----------|--------|
| [规则名1] | [可以引用原文的明确条件] | [field1] = [val] |
| [规则名2] | [明确条件] | [field2] = [值] |
| 边界情况 | [edge case 描述] | [输出] |
| 异常情况 | [error case 描述] | status = rejected |

---

## 四、前端 Parser 绑定说明

> prompt 改了，这里必须同步更新。parser 改了，也要回来对照这里。

| prompt 输出字段 | parser 解析逻辑 | UI 行为 |
|----------------|----------------|---------|
| [field1] = val_a | parser → state.xxx = 'A' | [显示 / 隐藏 / 跳转] |
| [field1] = val_b | parser → state.xxx = 'B' | [显示 / 隐藏 / 跳转] |
| [field2] >= 85 | parser → 高分模式 | [显示夸奖文案] |
| [field2] < 60 | parser → 低分模式 | [显示改进建议] |

**Fallback 策略**（主解析失败时）：
1. 先尝试正则精确匹配
2. 匹配失败 → 降级到位置解析
3. 位置解析也失败 → 返回默认值，不报错给用户

---

## 五、兜底校验

> 前端加这层校验，不能完全依赖 prompt 保证正确性。

矛盾检测规则：
- [field_A] 说「有」但 [field_B] 给了极低值 → 以 [field_B] 为准 / 触发 fallback
- [说明其他矛盾情况]

异常处理：
- AI 返回空内容 → [处理方式]
- 解析结果字段缺失 → [处理方式]
- 网络超时 → [处理方式]

---

## 六、测试用例

> 每次修改 prompt 后，跑一遍这些用例，对比改前改后的输出差异。

| 场景 | 输入 | 期望输出 | 
|------|------|----------|
| 正常情况 | [典型输入] | [期望字段值] |
| 边界情况 | [极端输入] | [期望字段值] |
| 故意写错 | [错误输入] | status = rejected |
| 空输入 | 空字符串 | [处理方式] |
```

---

## 六条核心原则（写 prompt 时的检查清单）

- [ ] 输出格式是否已定义，有无 example？
- [ ] Step Zero 门槛检查是否存在？
- [ ] 所有判断规则是否明确，无模糊措辞？
- [ ] AI 输出字段是否直接映射到 UI 行为，无中间翻译层？
- [ ] prompt 改动是否已同步通知 parser？
- [ ] 测试用例是否覆盖正常 / 边界 / 异常三类？
